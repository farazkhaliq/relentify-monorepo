Every property your agency manages has keys — and every key represents a security responsibility. Lose a key and you have a security breach. Issue a key without recording it and you have no accountability. Fail to collect keys at the end of a tenancy and you have an access control problem that may persist for years.

Key management sounds simple. In practice, it is one of the most common sources of operational frustration in a letting agency. Keys are physical objects that move between people — agents, tenants, contractors, landlords — and without a system to track their movement, they inevitably go astray.

## The scope of the problem

Consider the key-related events in a typical letting agency. A new tenant moves in and receives two sets of keys, a fob for the communal entrance, and a garage remote. A plumber needs access to fix a leak and borrows a set from the office. A landlord drops off a spare set. A tenant loses their keys and needs a replacement. A viewing is scheduled and the agent needs the keys to the vacant property.

Each of these events involves a key changing hands. Without a log of who has which keys at any given time, the agency has no way of knowing where its keys are — until someone needs one and it cannot be found.

## Building a key register

The foundation of key management is a key register — a record of every key held by the agency, which property it belongs to, and who currently has possession of it.

Your CRM or property management system should support a key register linked to each property record. For each property, the register should note the number and type of keys held (front door, back door, communal entrance, garage, postbox), the physical location of each set (key cabinet, issued to contractor, held by tenant), and the history of who has had possession.

When keys are issued — to a contractor for a maintenance visit, to an agent for a viewing, to a tenant at move-in — the register is updated. When keys are returned, the register is updated again. This creates an audit trail that answers the question "where are the keys to number 14?" at any given moment.

## Key issue and return process

Every key handover should follow a defined process. This does not need to be bureaucratic, but it does need to be consistent.

When keys are issued, record the date, the recipient, the reason, and the expected return date. Have the recipient sign for the keys — physically or digitally. If keys are issued to a contractor, note the job reference so the handover can be linked to the specific maintenance visit.

When keys are returned, record the date and confirm that all issued keys have been returned. If keys are not returned by the expected date, an automated reminder should prompt follow-up.

This process takes less than a minute per handover but prevents the vast majority of key-related problems.

## Tenant key management

At the start of a tenancy, the number and type of keys issued to the tenant should be documented in the tenancy agreement and in the CRM. At the end of the tenancy, all keys should be collected and checked against the original record.

If keys are not returned at the end of the tenancy, the cost of replacement or re-keying should be deducted from the deposit — but this is only defensible if you can demonstrate what was issued and what was not returned. The key register provides this evidence.

Lost keys during a tenancy are the tenant's responsibility, but the agent often needs to facilitate the replacement — arranging a locksmith, issuing replacement keys, and updating the register.

## Contractor access

Contractors need access to properties regularly — for maintenance, inspections, and viewings. Managing this access efficiently while maintaining security requires clear protocols.

For routine maintenance, the most common approach is to issue the contractor keys from the office and collect them when the job is complete. The key register tracks this handover.

For urgent maintenance, especially outside office hours, alternative arrangements may be needed — a key safe at the property, access via a digital lock, or coordination with the tenant to provide access directly.

Whatever the method, the CRM should record how access was provided for each maintenance visit. This audit trail protects the agency if a security issue arises.

## Key safes and digital access

Physical key management is increasingly being supplemented — or replaced — by technology. Key safes, electronic lockboxes, and smart locks all offer alternatives to traditional key handover.

Key safes are particularly useful for properties that need frequent access — vacant properties being marketed, properties with regular maintenance requirements. The combination can be changed after each use, and access can be granted remotely without anyone needing to visit the office.

Smart locks take this a step further, allowing access to be granted via a smartphone app. Access codes can be time-limited, ensuring that a contractor has access only during the scheduled visit. Every entry is logged electronically, creating a detailed access history.

If your agency uses these technologies, your CRM should integrate with them or at least record the access codes and entry logs alongside the property record.

## Security considerations

Key management is fundamentally a security function. The keys your agency holds provide physical access to your landlords' properties — properties that may contain valuable furnishings, personal belongings, or sensitive documents.

A key loss is a security incident and should be treated as one. If a set of office keys is lost, the affected properties should be re-keyed. If a contractor fails to return keys, follow-up should be immediate.

Key storage in the office should be secure — a locked key cabinet, ideally with restricted access. Keys should be labelled with a code rather than a property address, so that a lost key cannot be easily linked to a specific property by a third party.

## Reporting and compliance

Key management may be audited as part of a regulatory inspection or landlord review. A complete, up-to-date key register demonstrates that your agency takes security seriously and has proper controls in place.

Your CRM should be able to produce a report showing the current status of keys for any property — how many sets exist, where each set is, and the history of who has had possession. This report should be producible on demand, not after hours of searching.

Key management is a small but important part of running a professional letting agency. Get it right and you avoid the security breaches, the wasted time, and the client frustration that come from lost or untracked keys. Get it wrong and the consequences — both operational and reputational — can be disproportionately large.
