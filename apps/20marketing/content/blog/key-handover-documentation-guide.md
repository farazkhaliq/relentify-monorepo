Keys are one of the simplest items in a property inventory, but they cause a disproportionate number of headaches when not documented properly. Missing keys at check-out, disputes about how many sets were provided, charges for lock changes, and security concerns about unreturned keys are all common issues that proper documentation prevents.

A clear key handover process — documented as part of the inventory — takes minutes to set up and eliminates hours of confusion later.

## Why key documentation matters

### Security

Unreturned keys are a security risk. If a tenant moves out and does not return all keys, the landlord faces a choice: trust that the tenant will not use the keys, or change the locks. Changing locks is expensive and disruptive, but it may be necessary if there is any doubt about who has access to the property.

Proper documentation establishes exactly how many keys were issued, so any shortfall at check-out is immediately apparent.

### Deposit deductions

If keys are missing at check-out, the landlord may deduct the cost of replacement keys or lock changes from the deposit. This deduction needs to be evidenced — the landlord must show how many keys were provided and how many were returned. Without documentation, the deduction is difficult to support.

### Tenant peace of mind

Tenants benefit from key documentation too. If the check-out report claims a key is missing but the tenant returned everything they were given, the check-in record settles the matter.

### Regulatory compliance

In some jurisdictions, providing copies of all keys to the tenant at the start of the tenancy is a legal or contractual obligation. Documenting the handover provides proof of compliance.

## What to document at key handover

### Number of sets

Record the total number of key sets provided. A "set" typically includes all the keys needed to access the property and its various locks. If different keys are distributed separately (for example, one tenant gets the front door key and another gets the back door key in a shared house), document each individual key.

### Description of each key

For each key, record:
- What lock it operates (front door, back door, garage, shed, window locks, meter cupboard, mailbox, communal entrance)
- The key type (Yale, mortice, deadlock, electronic fob, code)
- Any identifying marks or labels
- Whether it is an original or a copy

### Electronic access

Many modern properties use electronic access systems:
- Key fobs for communal doors
- Access codes for keypads or smart locks
- Remote control for gates or garages
- Smartcard access for building entry

Document every electronic access device or code provided to the tenant, including serial numbers or identification marks on fobs and cards.

### Window keys

If the property has lockable windows (common with UPVC double glazing), record the number of window keys provided and which windows they operate. Window keys are frequently lost and rarely missed until the end of the tenancy.

### Garage and outbuilding keys

Keys for garages, sheds, storage units, and outbuildings should be documented separately from the main property keys.

### Photographs

Photograph all keys laid out together, clearly showing each one. Label them in the photograph if possible (laying them on a sheet of paper with written labels works well). This visual record complements the written description and makes the check-out comparison straightforward.

## Creating a key schedule

A key schedule is a simple document or section of the inventory that records all key information. It should include:

| Key | Type | Lock/Access | Quantity | Condition | Notes |
|-----|------|-------------|----------|-----------|-------|
| Front door | Yale | Main entrance | 2 | Working | Original keys |
| Front door | Mortice | Main entrance deadlock | 2 | Working | Brass finish |
| Back door | Yale | Kitchen exit | 1 | Working | Slightly stiff |
| Garage | Up-and-over | Detached garage | 1 | Working | Key marked "G" |
| Fob | Electronic | Communal entrance | 2 | Working | Serial: 4521, 4522 |
| Window keys | UPVC | All windows | 3 | Working | Generic type |

This schedule becomes part of the check-in inventory. At check-out, each item is checked against the list.

## The handover process

### At check-in

1. Assemble all keys, fobs, and access devices before the tenant arrives
2. Count and verify against the key schedule
3. Photograph the complete set
4. Hand over to the tenant
5. Have the tenant sign to acknowledge receipt (or record acknowledgment digitally)
6. Provide the tenant with a copy of the key schedule

### During the tenancy

- If the tenant requests additional keys (for a partner or family member), the landlord or agent should arrange cutting and document the additional keys
- If keys are lost during the tenancy, the tenant should report it immediately. Document the loss and any replacement keys or lock changes
- At mid-tenancy inspections, note whether the tenant still has all keys (a simple verbal check is usually sufficient)

### At check-out

1. Retrieve the check-in key schedule
2. Collect all keys from the tenant
3. Count and verify against the schedule
4. Note any missing keys
5. Test each key in its respective lock (keys can be returned but may not work if the wrong ones are returned)
6. Photograph the returned keys
7. Document in the check-out report

## Handling missing keys

### Assessing the risk

Not all missing keys carry the same risk. A missing window key is a minor inconvenience — a replacement can usually be obtained cheaply. A missing front door key raises security concerns that may necessitate a lock change.

Consider:
- How many copies of the key exist
- Whether the tenant is trustworthy (subjective, but relevant)
- Whether the lock can be rekeyed (cheaper than replacement)
- The type of property and security requirements

### Calculating the deduction

If locks need to be changed because of missing keys, the deduction should cover the actual cost:
- Cost of new lock and keys, or
- Cost of rekeying the existing lock, or
- Cost of replacing lost fobs or access devices

Get a quote or invoice from a locksmith to support the deduction. An unsupported claim for "lock change — 200" is harder to defend than "locksmith invoice attached — 185 inc. VAT."

### Common sense applies

If a tenant returns all keys except one window key that costs a few pounds to replace, applying a heavy-handed deduction is disproportionate and may be challenged. Be reasonable — the deduction should reflect the actual cost and risk, not a punitive charge.

## Spare keys and landlord access

### Landlord's copies

It is standard practice for the landlord or agent to retain a spare set of keys for emergency access. Document this in the tenancy agreement and inform the tenant. The spare set should be securely stored — not in an unlocked desk drawer.

### Key safes

For properties where tradespeople or agents need regular access (for maintenance, inspections, or showing to prospective tenants), a key safe can be used. Document the location of the key safe and who has the code.

### Agent key holding

If a letting agent holds keys on behalf of the landlord, this should be documented as part of the property management agreement. The agent should maintain a key log showing who has access to the keys and when they are collected and returned.

## The bottom line

Key documentation is a small but important part of property inventory management. It takes a few minutes at check-in and check-out, but it prevents security concerns, supports deposit deductions when keys go missing, and provides clarity for all parties.

Include a comprehensive key schedule in every inventory. Photograph every key. Get the tenant's acknowledgment. And at check-out, verify everything against the record.

It is one of the simplest things to get right — and one of the most frustrating to deal with when it goes wrong.
